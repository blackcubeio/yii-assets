# API

## Interfaces

### ManifestAssetBundleInterface

Contract for asset bundles that load files from a build manifest.

| Method | Description |
|--------|-------------|
| `getCatalogFile(): string` | Catalog filename relative to `sourcePath` (e.g. `assets-catalog.json`) |
| `loadFromCatalog(array $catalog): void` | Populate `$js` and `$css` arrays from decoded catalog data |

## Classes

### ManifestAssetLoader

Decorator for `AssetLoaderInterface` that handles manifest-based bundles. Works with all standard `AssetBundle` classes — adds manifest loading only for bundles implementing `ManifestAssetBundleInterface`.

```php
use Blackcube\Assets\ManifestAssetLoader;

// Registered via DI as AssetLoaderInterface
// Wraps the standard AssetLoader
$loader = new ManifestAssetLoader($innerLoader, $aliases);
```

When `loadBundle()` encounters a `ManifestAssetBundleInterface`, it reads the catalog JSON file from `sourcePath` and calls `loadFromCatalog()`.

### ViteAssetBundle

Base asset bundle for Vite-built assets. Reads `assets-catalog.json` generated by the Vite manifest plugin.

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `$catalogFile` | `string` | `'assets-catalog.json'` | Catalog filename |
| `$jsOptions` | `array` | `['type' => 'module', 'defer' => true]` | JS options (ES modules) |

```php
use Blackcube\Assets\ViteAssetBundle;

final class AppAsset extends ViteAssetBundle
{
    public ?string $basePath = '@assets';
    public ?string $baseUrl = '@assetsUrl';
    public ?string $sourcePath = '@assetsSource/dist-vite';
}
```

Catalog entries are loaded sequentially — each entry with a `js` or `css` key is added to the bundle.

### WebpackAssetBundle

Base asset bundle for Webpack-built assets. Reads `assets-catalog.json` generated by `assets-webpack-plugin`.

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `$catalogFile` | `string` | `'assets-catalog.json'` | Catalog filename |
| `$bundles` | `string[]` | `[]` | Explicit bundle order (empty = auto-detect) |
| `$cssOnly` | `string[]` | `[]` | Bundles that load only CSS (skip JS) |
| `$jsOnly` | `string[]` | `[]` | Bundles that load only JS (skip CSS) |
| `$jsPosition` | `?int` | `WebView::POSITION_HEAD` | JS position in page |
| `$jsOptions` | `array` | `['defer' => 'defer']` | JS options |

```php
use Blackcube\Assets\WebpackAssetBundle;

final class AppAsset extends WebpackAssetBundle
{
    public ?string $basePath = '@assets';
    public ?string $baseUrl = '@assetsUrl';
    public ?string $sourcePath = '@assetsSource/dist-webpack';
}
```

When `$bundles` is empty, all entries from the catalog are loaded with priority ordering: `manifest` first, then `vendors`, then the rest. Override `$bundles` for explicit control:

```php
final class AppAsset extends WebpackAssetBundle
{
    public ?string $basePath = '@assets';
    public ?string $baseUrl = '@assetsUrl';
    public ?string $sourcePath = '@assetsSource/dist-webpack';

    public array $bundles = ['manifest', 'vendors', 'app'];
    public array $jsOnly = ['manifest'];
}
```

### AssetsController

Console command that scaffolds Vite and/or Webpack build configuration.

```bash
./yii blackcube:assets/init
```

Prompts for builder selection, source directory, and output directory, then generates all necessary configuration files (`assets-blackcube.json`, `package.json`, `tsconfig.json`, build tool configs).

Registered automatically via config-plugin in `config/console/params.php`.
